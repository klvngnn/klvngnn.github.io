Week 3: Arrays and Hashes

2015/3/15

This week in phase0, I learned about Arrays and Hashes, and when to use them in different situations. The following describes and compares both; some examples are also included:

An Array can be thought of as a list; a collection of data in some kind of order. Any piece of data can be looked up by using the number corresponding to it on the list.

If had a list of things we needed to do, and we put them in an array:

things_to_do = ['code', 'yoga', 'eat']

We could look up the third thing by typing things_to_do[2] it would give us eat "But, Kelvin! You typed in 2 instead of 3, how does that give us the third thing?"

Well, because in an array, the first thing refered to is always 0. It's just how computers count things :-D

A hash can be thought of as a collection of pairs of information, and each pair is made of a name and a value. The name is called a key, and it must be unique. The key is used to return the value paired to it. This generally gives a more detailed view of items.

For example, in a shopping list, a hash could give you specifically what you need when you type in a generalized item:

shopping_list = {'fish' => 'Tilapia',

'milk' => '2%'

'sweets' => 'chocolate'

}

So in this hash, the keys are the generalized items, if you type in shopping_list['sweets'] it should return chocolate, nifty huh!

Both array and hash are similar in the fact that they both store and retrieve date.

They differ in the way that you use to look up and retrieve data. At least for the exercises that's how I feel that they differ mostly. One other thing is that Arrays seem to work better in compling and computing data.
