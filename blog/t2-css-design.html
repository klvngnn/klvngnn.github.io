Week 2: What are the differences between relative, absolute, and fixed positioning?

2015/3/8

One of the concepts that confused me the most this week were differences between relative, absolute, and fixed positioning of blocks. Here I go trying to organize my thoughts:

As blocks start out, they start out STATIC. All page elements start out as static. Static means the element will flow into the page as it normally would. The only reason you would set an element into static would be to forcefully-remove some positioning that got applied to an element outside of your control.

When blocks are changed to RELATIVE, the block's position moves relative to itself. If position was set to relative on an element, but no other positioning attributes (top, left, bottom or right) were used, there would be no effect on the element. It would as if it were STATIC. However, if other positioning attributes were introduced, for example, "top: 25px", the element would shift DOWN 10 pixels from where it would NORMALLY be. The ability to shift an element around based on it's regular position is very useful. It can be used to line up elements that don't want to line up.

When elements are positioned with ABSOLUTE, the element is placed exactly as dictated. Positioning attributes top, left bottom and right are used to set the location. These values are relative to the next parent element with relative (or absolute) positioning. If there is no such parent, it will default all the way back up to the element itself. Thus placing the element relatively to the page (Mostly to the top). The trade-off about absolute positioning is that these elements are removed from the flow of elements on the page. An element with this type of positioning is not affected by other elements and it doesn't affect other elements.

When FIXED is used, the element is positioned relative to the browser window itself. The element doesn't change when the window is scrolled, so a fixed positioned element will stay right where it is when the page is scrolled. The Pro of this positioning is that it keeps the navigation present at all times, and it creates an interesting effect on the page. The Cons are that there may be some usability concerns. On smallish laptops, the content may becut off and there is no way to scroll down to see the rest of the content.

Writing this out help me understand these concept better, and I hope those that are reading have understood this better also. Till next time. klvn out.
